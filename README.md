## Описание работы

Данная работа представляет собой программную реализацию расчёта возмущающих ускорений, возникающих из-за нецентральности гравитационного поля Земли. Расчёт выполняется для искусственного спутника Земли на эллиптической орбите с использованием разложения гравитационного потенциала в ряд по сферическим функциям.

**Цель работы:**
- Определение компонент возмущающего ускорения (S, T, W) в различных системах координат
- Анализ влияния гармоник геопотенциала на движение спутника
- Оценка отношения возмущающего ускорения к основному ньютоновскому ускорению
- Визуализация результатов расчётов и построение графиков зависимостей

## Теоретические основы

### Гравитационный потенциал Земли

Гравитационное поле Земли описывается потенциалом, который в сферических координатах представляется в виде:

```
U = μ/r + U_vozm
```

где:
- `μ = 398600.44 км³/с²` — гравитационный параметр Земли
- `r` — расстояние от центра Земли до точки
- `U_vozm` — возмущающий потенциал, обусловленный нецентральностью поля

Возмущающий потенциал раскладывается в ряд по сферическим функциям:

```
U_vozm = (μ/r₀) × Σ(n=2 to ∞) Σ(k=0 to n) (r₀/r)^(n+1) × Pₙᵏ(sin φ) × [Cₙₖ cos(kλ) + Sₙₖ sin(kλ)]
```

где:
- `r₀ = 6378.14 км` — экваториальный радиус Земли
- `φ` — широта
- `λ` — долгота
- `Pₙᵏ` — присоединённые полиномы Лежандра
- `Cₙₖ`, `Sₙₖ` — коэффициенты геопотенциала
- `Jₙ = -Cₙ₀` — зональные гармоники

### Компоненты возмущающего ускорения

Возмущающее ускорение вычисляется как градиент возмущающего потенциала и проецируется на три направления:

- **S** — радиальная компонента (по направлению от центра Земли)
- **T** — тангенциальная компонента (в плоскости орбиты, перпендикулярно S)
- **W** — бинормальная компонента (перпендикулярно плоскости орбиты)

Проекции вычисляются по формулам:

```
S = j_r
T = j_φ × cos(A) + j_λ × sin(A)
W = j_φ × sin(A) - j_λ × cos(A)
```

где `A` — азимут, определяемый через элементы орбиты.

## Архитектура проекта

Проект выполнен в виде **монорепозитория** и состоит из трёх основных модулей:

### 1. Backend (NestJS API)
RESTful API для выполнения расчётов, построенное на фреймворке NestJS с использованием TypeScript.

**Основные возможности:**
- Расчёт возмущающих ускорений по заданным элементам орбиты
- Поддержка различных систем координат: АГЭСК (ECI) и ГСК (ECEF)
- Учёт гармоник геопотенциала до 21-го порядка включительно
- Сравнение полного расчёта с упрощённой моделью (только J₂-гармоника)
- Экспорт результатов в форматах CSV и PDF
- Интеграция со Swagger для документации API
- Кэширование результатов для оптимизации производительности

### 2. Frontend (React/Vite UI)
Интерактивный веб-интерфейс для визуализации результатов расчётов.

**Технологии:**
- **React 19** — библиотека для создания пользовательского интерфейса
- **Vite** — современный сборщик с горячей перезагрузкой
- **Mantine UI** — компонентная библиотека для стилизации
- **Plotly.js** — построение интерактивных графиков
- **React Three Fiber** — 3D-визуализация орбиты
- **Zustand** — управление состоянием приложения

**Функционал:**
- Выбор одного из 27 предустановленных вариантов орбит
- Ручной ввод элементов орбиты
- Настройка параметров расчёта (количество точек, порядок гармоник)
- Построение графиков зависимостей ускорений от высоты
- Отображение отношения |j|/|g| для оценки значимости возмущений
- 3D-визуализация орбиты спутника
- Экспорт результатов в CSV и PDF

### 3. Shared Package
Общая библиотека констант, типов и вспомогательных функций.

**Содержит:**
- Физические константы (гравитационный параметр, радиус Земли и т.д.)

## Реализованный функционал

### Математическая модель и алгоритмы

Реализован полный цикл расчёта возмущающих ускорений согласно методическим указаниям:

#### 1. Решение уравнения Кеплера
Численное решение трансцендентного уравнения для эксцентрической аномалии:
```
E = M + e × sin(E)
```
Методом последовательных приближений с точностью ε = 10⁻¹² рад.

#### 2. Вычисление координат спутника
- Определение истинной аномалии через эксцентрическую
- Вычисление расстояния от центра Земли: `r = a × (1 - e × cos(E))`
- Переход к сферическим координатам в АГЭСК по формуле (6)

#### 3. Преобразование систем координат
Переход от инерциальной системы (АГЭСК/ECI) к связанной с Землёй (ГСК/ECEF):
```
r_ECEF = R_z(S(t)) × r_ECI
```
где `S(t) = ω_e × t` — угол поворота Земли.

#### 4. Полиномы Лежандра
Рекуррентное вычисление присоединённых полиномов Лежандра Pₙᵏ(x) и их производных.

#### 5. Возмущающие ускорения
Вычисление компонент ускорения в сферических координатах по формуле (11):
```
j_r = ∂U_vozm/∂r
j_φ = (1/r) × ∂U_vozm/∂φ
j_λ = (1/(r×cos(φ))) × ∂U_vozm/∂λ
```

#### 6. Проекции на орбитальную систему координат
Переход от сферических компонент к орбитальным (S, T, W) через азимут A.

### API endpoints

#### Основные расчётные методы

**POST `/api/calculate`**  
Выполнение полного расчёта возмущающих ускорений.

**Параметры запроса:**
```typescript
{
  orbit: {
    a: number;      // большая полуось, км
    e: number;      // эксцентриситет
    i: number;      // наклонение, град
    Omega: number;  // долгота восходящего узла, град
    omega: number;  // аргумент перицентра, град
    M: number;      // средняя аномалия, град
  },
  options?: {
    pointsCount?: number;     // количество точек на орбите (по умолчанию 100)
    maxHarmonicN?: number;    // максимальный порядок n гармоник (по умолчанию 4)
    maxHarmonicK?: number;    // максимальный порядок k гармоник (по умолчанию 3)
    includeJ2Only?: boolean;  // включить расчёт только с J₂ (по умолчанию false)
    tSeconds?: number;        // время для преобразования координат, с (по умолчанию 0)
    coordinateSystem?: 'ECI' | 'ECEF'; // система координат для расчёта
  }
}
```

**Ответ:**
```typescript
{
  success: true,
  data: {
    points: OrbitPoint[];     // массив точек орбиты
    summary: {
      minAcceleration: number;  // минимальное ускорение, м/с²
      maxAcceleration: number;  // максимальное ускорение, м/с²
      avgAcceleration: number;  // среднее ускорение, м/с²
      period: number;           // период обращения, с
    };
    constants: {
      mu: number;      // гравитационный параметр
      r0: number;      // радиус Земли
      harmonics: [];   // использованные гармоники
    };
  },
  executionTime: number;  // время выполнения, мс
}
```

**POST `/api/compare`**  
Сравнение результатов для двух наборов параметров орбиты.

#### Пресеты и константы

**GET `/api/presets`**  
Получение списка всех 27 предустановленных вариантов орбит.

**GET `/api/presets/:id`**  
Получение конкретного варианта орбиты (1-27).

**GET `/api/constants`**  
Получение физических констант и коэффициентов геопотенциала.

#### Экспорт данных

**POST `/api/export/csv`**  
Экспорт результатов расчёта в формате CSV.

**POST `/api/export/pdf`**  
Экспорт результатов расчёта в формате PDF с графиками.

#### Документация

**GET `/api/docs`**  
Swagger UI с полной документацией API.

## Быстрый старт (dev)

### Требования
- **Node.js** версии 18.x или выше
- **npm** версии 9.x или выше
- **Docker** и **Docker Compose** (опционально)

### Установка зависимостей

В корневой директории проекта выполните:

```bash
npm install
```

Эта команда установит зависимости для всех рабочих пространств монорепозитория:
- `@lab/api` — backend API
- `@lab/web` — frontend приложение
- `@lab/shared` — общая библиотека

### Запуск Backend (API)

```bash
npm run dev:api
```

API будет доступно по адресу: `http://localhost:3000`

Документация Swagger: `http://localhost:3000/api/docs`

### Запуск Frontend (Web UI)

```bash
npm run dev:web
```

Веб-интерфейс откроется по адресу: `http://localhost:5173`

### Конфигурация API для Web

По умолчанию веб-приложение ожидает API на `http://localhost:3000/api`.

Если API запущен на другом адресе, настройте через переменную окружения:

```bash
VITE_API_BASE="http://your-api-url/api" npm run dev -w @lab/web
```

### Доступные скрипты

```bash
# Запуск обоих приложений одновременно (в разных терминалах)
npm run dev         # только web
npm run dev:api     # только api
npm run dev:web     # только web

# Сборка всех проектов
npm run build       # сборка api, web и shared

# Проверка кода
npm run lint        # запуск ESLint для всех проектов

# Тестирование
npm run test        # запуск unit-тестов
npm run test:e2e    # запуск end-to-end тестов
```

Проект поддерживает контейнеризацию через Docker Compose для развёртывания в production-среде.

```bash

```

Эта команда:
1. Создаст образы для API и Web приложений
2. Запустит контейнеры с необходимыми зависимостями
3. Настроит сеть между контейнерами

После запуска:
- **Web интерфейс**: `http://localhost:8080`
- **API**: `http://localhost:3000/api`
- **Swagger документация**: `http://localhost:3000/api/docs`

### Остановка контейнеров

```bash
docker compose down
```

### Пересборка без кэша

```bash
docker compose build --no-cache
docker compose up
```

## Результаты и выводы

### Основные результаты работы

В ходе выполнения лабораторной работы было реализовано программное обеспечение для расчёта возмущающих ускорений, обусловленных нецентральностью гравитационного поля Земли.

**Что было сделано:**

1. **Изучены теоретические основы**
   - Разложение гравитационного потенциала в ряд по сферическим функциям
   - Методы вычисления зональных, тессеральных и секториальных гармоник
   - Формулы для определения компонент возмущающего ускорения

2. **Реализованы математические алгоритмы**
   - Численное решение уравнения Кеплера
   - Вычисление координат спутника в различных системах координат
   - Расчёт полиномов Лежандра и их производных
   - Определение возмущающих ускорений по формуле (11)
   - Проецирование ускорений на орбитальную систему координат

3. **Разработано приложение с архитектурой монорепозитория**
   - Backend на NestJS с REST API
   - Frontend на React с интерактивной визуализацией
   - Общий пакет констант и типов
   - Контейнеризация через Docker

4. **Проведены расчёты**
   - Для 27 вариантов орбит из методических указаний
   - Построены графики зависимостей ускорений от высоты
   - Оценено отношение возмущающего ускорения к ньютоновскому
   - Выполнено сравнение полного расчёта с моделью только J₂-гармоники

### Научная новизна и практическая значимость

**Научная новизна:**
- Комплексный учёт гармоник геопотенциала высокого порядка (до n=21)
- Возможность анализа влияния отдельных гармоник на движение спутника
- Визуализация результатов в реальном времени

**Практическая значимость:**
- Полученные результаты могут быть использованы для:
  - Уточнения параметров движения искусственных спутников Земли
  - Прогнозирования эволюции орбит
  - Планирования космических миссий
  - Обучения студентов методам небесной механики

### Выводы

1. Возмущающие ускорения, обусловленные нецентральностью гравитационного поля Земли, составляют величину порядка 10⁻³–10⁻⁶ м/с² в зависимости от высоты орбиты.

2. Отношение |j|/|g| показывает, что возмущения наиболее значимы на низких орбитах и уменьшаются с увеличением высоты.

3. Зональная гармоника J₂ (сжатие Земли) даёт основной вклад в возмущающее ускорение, однако для точных расчётов необходимо учитывать и другие гармоники.

4. Разработанное программное обеспечение позволяет выполнять расчёты с высокой точностью и наглядно представлять результаты.
